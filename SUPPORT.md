# Support

Use the following paths so questions and reports stay actionable:

- Bug reports: open a GitHub issue with reproduction steps, expected behavior, and relevant logs with secrets redacted.
- Feature requests: open a GitHub issue describing the problem, proposed solution, and alternatives considered.
- Security reports: follow the private reporting guidance in [`SECURITY.md`](./SECURITY.md).

Before opening an issue:

- Read [`README.md`](./README.md) for the project overview and local development entry points.
- Read [`GET-STARTED.md`](./GET-STARTED.md) for the local regtest workflow.
- Check [`CONTRIBUTING.md`](./CONTRIBUTING.md) for development and testing expectations.
